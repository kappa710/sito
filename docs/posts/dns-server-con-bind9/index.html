fonti : https://www.digitalocean.com/community/tutorials/how-to-configure-bind-as-a-private-network-dns-server-on-ubuntu-20-04 e 3. Configurations and Zone Files — BIND 9 9.20.9 documentation

installa bind 9

sudo apt install bind9 bind9utils bind9-doc

devi andare nelle cartella /etc/bind

aprire il file named.conf.options e scrivi qualcosa del genere

options{

forwarders{
	8.8.8.8;
	1.1.1.1;
	}
};

configura il file named.conf.local



zone "nome.zona" {
    type primary;
    file "/etc/bind/zones/db.nome.zona"; # zone file path

};

creare cartella zones e poi creare il file db.nome.zona e in questo file

$TTL 86400  ; TTL predefinito: 1 giorno (86400 secondi)

@       IN      SOA     ns1.nome.zona. hostmaster.nome.zona. (
                              2025060701 ; Serial: Numero di serie (AnnoMeseGiornoSequenza)
                              7200       ; Refresh: 2 ore
                              3600       ; Retry: 1 ora
                              604800     ; Expire: 1 settimana
                              86400 )    ; Negative Cache TTL: 1 giorno

; Record NS (Name Server): Indica chi è il server autorevole per il dominio
@       IN      NS      ns1.nome.zona.

; Record A (Address): Mappa i nomi agli indirizzi IPv4
@       IN      A       192.0.2.100    ; Indirizzo IP del dominio base (nome.zona)
ns1     IN      A       192.0.2.101    ; Indirizzo IP del tuo name server (ns1.nome.zona)
www     IN      A       192.0.2.100    ; Indirizzo IP per www.nome.zona (stesso del dominio base)
mail    IN      A       192.0.2.102    ; Indirizzo IP del tuo server di posta (mail.nome.zona)

; Record MX (Mail Exchanger): Specifica il server per la posta elettronica
@       IN      MX 10   mail.nome.zona. ; Il tuo server di posta, con priorità 10

ATTENTO A METTERE SEMPRE I PUNTI ALLA FINE

SECONDA RIGA @ si riferisce a dominio nome.zona IN SOA {MNAME = nome server principale per zona se creo altri server secondari per zona sempre questo deve stare } {email di gestice}

TERZA RIGA differenza TERZA RIGA VS SECONDA RIGA terza riga usata dal mondo esterno , seconda riga usata da server secondari scelti da noi per capire chi è il principale in “locale”

serve dare ip a ns1 se no esterno non sa come collegarci al nostro server

per restartare bind9 dopo modifica

sudo systemctl restart bind9
